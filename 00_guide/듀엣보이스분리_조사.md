# Suno AI 듀엣 보이스 분리 조사

## 결론

**Suno AI의 한계가 주된 원인이며, 현재 프롬프트 방식도 개선 여지가 있다.**

듀엣은 Suno의 공식 기능이 아니라 "exploit(편법)"에 가깝다. 보이스 태그는 "hit and miss"이며, 성별이 뒤바뀌거나 두 목소리가 너무 비슷하게 나오는 일이 빈번하다.

---

## 1. Suno AI 자체 한계 (주된 원인)

- 듀엣은 공식 기능이 아님 — 보이스 분리를 보장하지 않는다
- `[Male]`, `[Female]` 태그는 힌트일 뿐, 100% 반영되지 않는다
- 성별이 뒤바뀌는 현상 빈번 (Male 파트를 Female이 부르거나 그 반대)
- 두 보이스가 너무 유사하게 생성되는 경우도 많음
- 같은 프롬프트로 여러 번 생성해야 원하는 결과를 얻을 수 있다

## 2. 현재 프롬프트의 개선 가능 포인트

### Style of Music 프롬프트
- **현재**: `Male and Female duet` 포함 (기본적으로 맞음)
- **개선**: 보컬 타입을 구체적으로 명시하면 분리 확률 상승
  - 예: `Pop Duet featuring male tenor and female soprano`
  - 장르에 `Duet`을 반드시 포함 (예: `K-Pop Ballad Duet`)

### 가사 태그 형식
- **현재 방식**: `[Male, Conversational]`, `[Female, Sharp]`
- **권장 방식**: 섹션명에 보이스를 결합
  - `[Verse 1: Male Vocal]`
  - `[Verse 2: Female Vocal]`
  - `[Chorus: Male and Female Duet]`
- 태그는 **짧게 1~3단어**가 가장 효과적
- 모든 줄에 `[Female]`, `[Male]`, `[Both]`를 명시하면 분리 확률 상승

### 페르소나(이름) 사용법
- 성별 태그 대신 이름을 사용하면 분리가 더 잘 됨
  - `[Verse 1: Gabriel]` / `[Verse 2: Rebecca]` / `[Chorus: Both]`
- 다만 이름을 써도 뒤바뀌는 현상은 여전히 존재

### 구조적 팁
- 듀엣 섹션은 8~12줄 이내로 유지
- `[Both]`는 짧은 훅 한 줄에만 사용, 두 사람이 동시에 긴 문장을 부르지 않기
- 오버랩은 짧은 훅, 에코, 애드리브 정도로 제한
- 뮤지컬/브로드웨이 장르가 듀엣 분리에 가장 유리

## 3. 현실적인 대응 방법

| 방법 | 설명 |
|------|------|
| 다회 생성 | 같은 프롬프트로 여러 번 생성해서 보이스 분리가 잘 된 버전 선택 |
| 장르 활용 | Musical Theater, Broadway 등 듀엣이 일반적인 장르 사용 |
| Studio 수정 | Suno V5 Studio에서 섹션별로 보이스 수정 후 스템 내보내기 |
| 성별 중립 가사 유지 | 보이스가 뒤바뀌어도 어색하지 않게 작성 (현재 규칙 유지) |

## 4. 요약

| 구분 | 현재 | 권장 |
|------|------|------|
| Style | `Male and Female duet` | `K-Pop Ballad Duet featuring male tenor and female soprano` |
| 가사 태그 | `[Male, Conversational]` | `[Verse 1: Male Vocal]` |
| Both 사용 | Chorus 전체 | 짧은 훅 1줄만 |
| 기대치 | 1회 생성으로 완벽 분리 | 2~3회 생성 후 선별 필요 |
