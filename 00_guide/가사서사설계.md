# 가사 서사 설계

> 3단계(Suno 프롬프트 생성) **직전에** 실행한다.
> 가사를 쓰기 전에 곡의 서사 뼈대를 먼저 세운다.
> 서사 없이 이미지부터 쌓으면 "같은 상태의 다른 묘사"가 반복되고 시간이 흐르지 않는다.

---

## 왜 서사가 필요한가

가사는 3분 안에 화자가 A 지점에서 B 지점으로 이동하는 기록이다.
서사가 없으면 Verse 1과 Verse 2가 같은 상태를 다른 오브제로 반복 묘사하게 된다.
분위기 중심 장르(다크신스, 앰비언트, 드림팝 등)도 서사가 있어야 청자가 곡 안에서 이동감을 느낀다.

- 서사 = 시간의 흐름. "처음에 이랬는데 끝에 저렇게 되었다"
- 서사 ≠ 스토리텔링. 사건이 필요한 게 아니라, 화자의 내면 위치가 이동하면 된다

---

## 1. 화자 설정

가사를 쓰기 전에 노래를 부르는 사람이 누구인지 정한다.

| 항목 | 설명 | 예시 |
|------|------|------|
| **누구인가** | 나이, 성별, 직업/역할은 필요할 때만. 핵심은 "이 사람이 왜 이 노래를 부르는가" | 자정에 라디오를 틀어놓고 잠 못 드는 사람 |
| **어디에 있는가** | 물리적 공간. 곡 전체의 배경이 된다 | 어두운 방, 라디오 앞 |
| **지금 무엇을 하고 있는가** | 곡이 시작되는 순간의 행동 | 다이얼을 돌리고 있다 |

화자가 모호하면 가사가 "누구의 것도 아닌 관찰"이 된다.
화자가 명확하면 같은 이미지라도 그 사람의 시선으로 필터링된다.

---

## 2. 화자의 상태 (심리·감정)

화자가 곡의 시작과 끝에서 어떤 심리 상태인지 정한다.
**시작 상태와 끝 상태가 달라야** 서사가 생긴다.

| 항목 | 설명 |
|------|------|
| **시작 감정** | 곡이 열리는 순간 화자의 내면. Verse 1의 톤을 결정한다 |
| **끝 감정** | 곡이 닫히는 순간 화자의 내면. Final Chorus/Outro의 톤을 결정한다 |
| **변화의 계기** | 시작→끝 사이에 무엇이 감정을 바꾸는가. Bridge 또는 Verse 2에서 일어난다 |

감정 변화 유형:

| 유형 | 시작 → 끝 | 예시 |
|------|----------|------|
| **저항→수용** | 거부하던 것을 받아들임 | 주파수를 끊으려 했지만 결국 듣기로 선택 |
| **확신→균열** | 믿고 있던 것에 금이 감 | 정해진 대로 살았다고 믿었는데 신호등이 꺼짐 |
| **무감→자각** | 느끼지 못하던 것을 느끼기 시작 | 컨베이어 위에서 아무것도 못 느끼다가 통증을 자각 |
| **고립→연결** | 혼자라고 생각했는데 연결됨 | 아무도 없는 거리에서 누군가의 흔적 발견 |
| **연결→고립** | 연결되어 있다고 믿었는데 끊어짐 | 택시 안에서 도시와 함께라고 느꼈는데 내려야 함 |
| **통제→상실** | 내가 조종한다고 생각했는데 조종당하고 있었음 | 다이얼을 돌리는 게 내가 아니었음 |
| **상실→재구성** | 잃어버린 뒤 다른 것을 발견 | 끊긴 신호 속에서 새로운 주파수를 찾음 |

---

## 3. 핵심 주제 (전하고 싶은 메시지)

곡이 끝난 뒤 청자에게 남아야 하는 **한 문장**.
가사에 직접 쓰지 않는다 — 직접 쓰면 설교가 된다.
모든 이미지와 서사가 이 한 문장을 향해 수렴해야 한다.

| 항목 | 설명 |
|------|------|
| **한 문장 주제** | "이 곡은 ___에 대한 곡이다"를 한 문장으로 |
| **가사에 쓰면 안 되는 이유** | 주제를 직접 말하면 청자가 해석할 여지가 사라진다. 주제는 소재를 통해 우회해야 한다 |

작성 테스트: 주제 문장을 가사의 아무 줄과 바꿔 넣었을 때, 그 줄이 더 나빠지면 올바른 주제.
주제를 직접 넣었을 때 오히려 가사가 좋아진다면, 소재가 주제를 충분히 담지 못하고 있다는 신호.

---

## 4. 소재 (주제를 우회하는 시적 재료)

핵심 주제를 직접 말하지 않고 표현하기 위한 구체적 사물·공간·행동.
소재는 장르의 세계관에서 가져와야 한다 (`00_guide/02_장르소재매칭분석.md` 참조).

| 항목 | 설명 |
|------|------|
| **중심 소재** | 곡 전체를 관통하는 하나의 오브제/공간. 제목이 되는 경우가 많다 |
| **보조 소재** | 중심 소재를 보강하는 2~3개의 세부 사물. Verse마다 다른 보조 소재를 쓰면 시간 진행 효과 |
| **이중성 설계** | 소재가 곡의 표면(리터럴)과 이면(비유) 양쪽으로 읽히는가 |

좋은 소재의 조건:
- 장르의 사운드와 같은 세계 출신이다 (칩튠 곡에 게임 용어, 다크신스에 산업/기계 용어)
- 물리적으로 만질 수 있다 (추상 명사 ✕ → 구체 사물 ○)
- 상태 변화가 가능하다 (켜짐/꺼짐, 열림/닫힘, 녹음/굳음 — 서사의 시간축이 된다)

## 5. 서사 곡선 — 섹션별 배치

위 4가지가 정해지면, 각 섹션에 화자의 위치를 배정한다.

```
Verse 1  — 시작 상태. 화자가 소재와 처음 만나는 순간.
           아직 "밖에 있는 것"으로 인식. 관찰자 시점.

Pre-Ch 1 — 균열. 무언가 예상과 다르다는 첫 신호.

Chorus   — 핵심 감정의 첫 분출. 아직 완전히 이해하지 못한 채 느낌만 터짐.

Verse 2  — 변화. 시작 상태에서 이동. 소재와의 거리가 좁혀짐.
           Verse 1과 같은 상태를 반복하면 안 됨.
           시간이 흘렀거나, 시도를 했거나, 무언가가 달라져야 함.

Pre-Ch 2 — 균열 확대. 더 이상 무시할 수 없음.

Chorus   — 같은 가사라도 Verse 2를 거쳤으므로 의미가 달라짐.

Bridge   — 전환점. 화자가 무언가를 깨닫거나, 선택하거나, 포기함.
           곡에서 가장 조용하거나 가장 다른 순간.

Final Ch — 끝 상태. 시작과 다른 곳에 서 있음.
           Chorus와 가사가 같아도 됨 — 단, 화자의 위치가 달라졌으므로 같은 말이 다르게 들려야 함.
           또는 가사를 변형하여 변화를 명시.
```

### 섹션 배치 자가 점검

| 점검 | 방법 |
|------|------|
| Verse 1과 Verse 2의 화자 위치가 다른가? | 두 Verse의 첫 줄만 비교. 같은 상태면 실패 |
| Bridge에서 무엇이 바뀌는가? | Bridge 전후로 화자의 태도가 반전되는가? 같으면 실패 |
| Final Chorus가 첫 Chorus와 같은 무게인가? | 같은 가사라도 다르게 들려야 함. 완전히 같으면 서사 부재 |

## 작성 형식

서사 설계는 Suno 프롬프트 파일(`04_Suno_Prompt/`)에 직접 쓰지 않는다.
가사를 쓰기 전에 내부적으로 수행하고, 결과물인 가사만 프롬프트에 반영한다.

설계 시 아래 항목을 순서대로 채운다:

```
화자: (누구, 어디서, 무엇을 하고 있는가)
시작 감정: (Verse 1의 톤)
끝 감정: (Final Chorus의 톤)
변화 계기: (Bridge에서 무엇이 바뀌는가)
핵심 주제: (가사에 직접 쓰지 않는 한 문장)
중심 소재: (곡을 관통하는 오브제)
보조 소재: (Verse마다 다른 2~3개)
이중성: (소재의 리터럴 의미 / 비유 의미)
```

이 설계가 끝난 뒤 가사를 쓴다. 가사를 쓰다가 서사에서 벗어나면 설계로 돌아와 점검한다.
