# Suno AI 프롬프트 생성 절차

이 문서는 음악유형과 음악컨셉을 조합하여 Suno AI Custom Mode에 입력할 Title, Style, Lyrics를 생성하는 전체 절차를 기술한다.

---

## 절차 요약

```
[0단계] 장르-소재 매칭 점검 (00_guide/02_장르소재매칭분석.md)
    ↓
[1단계] 음악유형 확인/선택
    ↓
[2단계] 음악컨셉 확인/선택
    ↓
[3단계] 음악유형 + 음악컨셉 조합
    ↓
[4단계] Suno 프롬프트 생성 (Title / Style / Lyrics)
    ↓
[5단계] 프롬프트 파일 저장 (04_Suno_Prompt)
```

---

## 0단계: 장르-소재 매칭 점검

음악유형과 컨셉을 조합하기 전에 **`00_guide/02_장르소재매칭분석.md`**를 참조한다.

| # | 점검 항목 | 질문 |
|---|----------|------|
| 1 | 출신 일치 | 장르가 태어난 문화와 가사 소재가 같은 세계인가? |
| 2 | 이중성 | 소재 용어가 일상 감정으로도 자연스럽게 읽히는가? |
| 3 | 교환 불가 | 이 가사를 다른 장르 곡에 넣어도 성립하는가? (성립하면 고유성 부족) |
| 4 | 감정 동기화 | 음악유형 분위기 키워드가 가사 감정과 같은 방향인가? |
| 5 | 후크-구조 일치 | 후크의 길이·반복이 장르 특성과 맞는가? |
| 6 | 에너지 곡선 | 가사 감정 흐름이 곡 구조 에너지와 동기화되는가? |

---

## 1단계: 음악유형 확인/선택

### 기존 유형이 있는 경우
1. `02_MusicType` 폴더의 파일 목록을 확인한다
2. 사용자가 원하는 용도에 맞는 음악유형 파일을 선택한다
3. 해당 파일의 장르, 악기, 템포, 분위기, 프로덕션, 제외 요소를 읽는다

### 기존 유형이 없는 경우 (신규 생성)
1. 사용자에게 용도(청취 목적)를 확인한다 (예: 운동용, 수면용, 카페용 등)
2. `01_음악유형.md`의 작성 규칙에 따라 새 파일을 생성한다
   - 파일명: `번호_용도명.md` (번호는 기존 파일 다음 순번)
   - 포함 항목: 특징(10가지), 장르, 악기, 템포, 분위기, 보컬, 곡 구조, 프로덕션, 제외 요소
3. 생성한 파일을 사용자에게 확인받는다

---

## 2단계: 음악컨셉 확인/선택

### 기존 컨셉이 있는 경우
1. `03_MusicConcept` 폴더의 파일 목록을 확인한다
2. 사용자가 원하는 주제·분위기에 맞는 컨셉 파일을 선택한다
3. 해당 파일의 주제, 소재, 분위기, 가사 주제, 사운드 힌트를 읽는다

### 기존 컨셉이 없는 경우 (신규 생성)
1. 사용자에게 곡의 주제·상황·감정을 확인한다 (예: 비 오는 날, 새벽 작업 등)
2. `01_음악컨셉.md`의 작성 규칙에 따라 새 파일을 생성한다
   - 파일명: `번호_컨셉명.md` (번호는 기존 파일 다음 순번)
   - 포함 항목: 설명, 주제, 소재, 분위기, 가사 주제, 사운드 힌트, 적합한 음악유형
3. 생성한 파일을 사용자에게 확인받는다

---

## 3단계: 음악유형 + 음악컨셉 조합

선택된 음악유형과 음악컨셉의 정보를 조합하여 Suno 입력값을 구성한다.

| 항목 | 음악유형에서 가져올 정보 | 음악컨셉에서 가져올 정보 |
|------|--------------------------|--------------------------|
| Style | 장르, 악기, 템포, 분위기, 프로덕션, 제외 요소 | 분위기, 사운드 힌트 |
| Lyrics | 보컬 스타일, 곡 구조 | 주제, 소재, 가사 주제 |
| Title | - | 주제, 소재에서 영감 |

### 조합 규칙
- Style: 음악유형의 장르가 베이스, 음악컨셉의 분위기·사운드 힌트를 추가
- 분위기 키워드가 중복되면 하나만 사용
- 제외 요소는 Style 맨 끝에 배치
- 보컬 유무는 음악유형의 보컬 항목을 따른다

---

## 4단계: Suno 프롬프트 생성

### 4.1 Title 생성
- 음악컨셉의 주제·소재를 반영한 영어 제목
- 간결하고 기억에 남는 형태 (1~4단어)

### 4.2 Style of Music 생성
- `02_Suno_입력필드_가이드.md`의 7요소 공식을 따른다
- **`00_guide/01_음악프롬프트사운드단어100개.md`의 사운드 단어를 적극 활용** (악기/리듬/질감/보컬/분위기 5개 카테고리)
- 형식: `[장르], [템포] BPM, [분위기], [악기], [보컬 스타일], [프로덕션], [제외 요소]`
- 4~8개 태그 적정
- 서술형으로 작성 (명령형 금지)

### 4.3 Lyrics 생성
- 음악유형의 곡 구조를 따라 구조 태그 배치
- 음악컨셉의 가사 주제를 반영하여 가사 작성
- **`00_guide/일본카피도감_분석.md`의 기법을 적용하여 가사 품질 향상** (아래 4.3.1 참조)
- Instrumental인 경우 `[Instrumental]`만 기입
- 보컬 전달 태그를 적절히 조합 (예: `[Verse 1] [Soft]`)

#### 4.3.1 카피도감 기법 적용
가사 작성 시 아래 기법을 의식적으로 활용한다. 참조: `00_guide/일본카피도감_분석.md`

| 적용 위치 | 기법 | 예시 |
|-----------|------|------|
| 후렴 핵심 구절 | **모순 결합** — 어울리지 않는 단어를 붙여 인상 강화 | "귀찮은 행복", "달콤한 한숨" |
| Verse 도입부 | **조건-귀결형** — 조건 제시 후 의외의 결과 | "아버지가 되면, 사진은 훌륭해진다" |
| Bridge / 전환부 | **질문형** — 청자를 끌어들이는 질문으로 마무리 | "당신의 차에는 두근거림이 있습니까?" |
| 가사 마지막 줄 | **결론 생략** — 여운을 남기고 끊기 | "사랑이 끝나버릴 거라면, 여름이 좋다" |
| 감정 강조 구간 | **구체적 숫자 + 추상적 감정** | "40번의 내일", "10대의 노래" |
| 전체 | **한 줄에 하나의 감정만** — 명사 종결 선호 | "마음에, 모험을." |

### 4.4 출력 형식

```
## Title
(곡 제목)

## Style of Music
(Suno Style 필드에 복붙할 내용)

## Lyrics
(Suno Lyrics 필드에 복붙할 내용 - 구조 태그 포함)
```

## 5단계: 프롬프트 파일 저장

생성된 Suno 프롬프트를 `04_Suno_Prompt` 폴더에 개별 md 파일로 저장한다.

### 파일 규칙
- 파일명: `번호_곡제목.md` (예: `01_Midnight_Echo.md`, `02_Rainy_Groove.md`)
- 번호는 01부터 순차 부여
- 곡 제목은 영어, 띄어쓰기는 언더스코어(`_`)로 대체

### 파일 내용 형식

```markdown
# (곡 제목)

- 음악유형: (사용한 음악유형 번호·이름)
- 음악컨셉: (사용한 음악컨셉 번호·이름)
- 생성일: (YYYY-MM-DD)

## Title
(Suno Title 필드에 복붙할 내용)

## Style of Music
(Suno Style 필드에 복붙할 내용)

## Lyrics
(Suno Lyrics 필드에 복붙할 내용 - 구조 태그 포함)
```

### 저장 후
- 사용자가 Suno에 각 섹션을 복붙하여 음악을 생성한다
- 생성 결과가 마음에 들지 않으면 Style이나 Lyrics를 수정하여 같은 파일을 업데이트한다
